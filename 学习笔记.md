## Day 1：Python 调用大模型 API

### 必须记住

| 序号 | 知识点 | 为什么重要 |
|------|--------|-----------|
| 1 | `messages = [{"role": "user", "content": "..."}]` | 和 LLM 通信的唯一格式 |
| 2 | `role` 有三种：system / user / assistant | 后面工具调用还会加第四种 tool |
| 3 | `response.choices[0].message.content` | 取 AI 回复的固定写法 |
| 4 | `.env` 存密码，`load_dotenv()` 读密码 | 每个项目都这套路 |
| 5 | Python 行首空格不能乱加 | 否则一直卡 IndentationError |

### 核心概念

```
messages = 对话记录列表
  └── {"role": "谁说的", "content": "说什么"}

response = 整个响应对象（快递箱）
  .choices[0] = 第一个候选回复（打开箱子）
    .message = 消息对象（取出内盒）
      .content = 真正的文本内容（你要的东西）
```

### 关键代码

```python
from dotenv import load_dotenv
from openai import OpenAI
import os

load_dotenv()

client = OpenAI(
    api_key=os.getenv("OPENAI_API_KEY"),
    base_url=os.getenv("OPENAI_BASE_URL"),
)

response = client.chat.completions.create(
    model=os.getenv("OPENAI_MODEL"),
    messages=[
        {"role": "system", "content": "你是一只猫，每句话结尾加喵"},
        {"role": "user", "content": "你好，1+1等于几"},
    ],
)

print(response.choices[0].message.content)
```

### 记忆口诀

> messages 是对话，choices[0] 是回复，content 是文字，其他全是包装盒。

## Day 2：安全计算器工具

### 必须记住

| 序号 | 知识点 | 为什么重要 |
|------|--------|-----------|
| 1 | `eval(str)` 能把任何字符串当代码执行 | 强大但危险 |
| 2 | `compile(str, "", "eval")` 编译但不执行 | 先安检后放行 |
| 3 | `code.co_names` 是字节码里引用的所有变量名 | 白名单检查的依据 |
| 4 | `{"__builtins__": {}}` 清空所有内置函数 | 最后一层防线 |
| 5 | `SyntaxError` 是编译失败抛的异常 | 拦截 import/print 等语句 |

### 核心概念：四层沙箱

```
用户输入 → 白名单过滤 → compile 检查语法 → co_names 检查名字 → 空 builtins 执行
```

### 关键代码

```python
import math

def safe_calc(expression: str) -> str:
    allowed = {k: v for k, v in math.__dict__.items()
               if not k.startswith("_")}
    allowed["abs"] = abs
    allowed["round"] = round
    code = compile(expression, "<calc>", "eval")
    for name in code.co_names:
        if name not in allowed:
            return f"错误：不允许使用 '{name}'"
    result = eval(code, {"__builtins__": {}}, allowed)
    return f"{expression} = {result}"
```

### 测试结果

```
"2 + 3 * 4"         → 14        [正常计算]
"sqrt(144) + 10"    → 22.0      [正常计算]
"__import__('os')"  → 被拦截     [危险操作]
"open('secret.txt')"→ 被拦截     [危险操作]
"print('hello')"    → 被拦截     [非法语句]
```

### 记忆口诀

> compile 先安检，co_names 查身份证，空 builtins 没武器，四层沙箱保平安。

## Day 3：让 AI 自动调用工具（Function Calling）

### 必须记住

| 序号 | 知识点 | 为什么重要 |
|------|--------|-----------|
| 1 | `tools` 参数传给 `create()` | 告诉 AI"你有这些工具可用"——Agent 的核心机制 |
| 2 | `msg.tool_calls` 不是文字回复 | AI 说"我要调工具"而不是直接给答案 |
| 3 | `tool_call.function.name` 和 `arguments` | AI 指定工具名和参数——按它说的执行 |
| 4 | `role: "tool"` 是第四种消息角色 | OpenAI 协议要求工具结果必须是 tool 角色 |
| 5 | 两轮对话：第一轮决策 + 第二轮总结 | Agent 的本质是"对话循环" |

### 核心概念：Agent 的两轮对话

```
         第一轮                          第二轮
┌──────────────────────┐    ┌──────────────────────┐
│ user: "35度等于多少F"  │    │ user: "35度等于多少F"  │
│                       │    │ assistant: 我要调计算器│
│ DeepSeek 看到 tools    │    │ tool: 35*9/5+32 = 95 │
│ 它决定: 调 calculate   │    │                       │
│ 返回: tool_calls       │    │ DeepSeek 看到结果      │
│                       │    │ 回复: "等于95F"        │
└──────────────────────┘    └──────────────────────┘
```

### 四种 role 完整表

| role | 谁 | 什么时候出现 |
|------|----|-------------|
| `system` | 你 | 规则/工具说明 |
| `user` | 用户 | 提问 |
| `assistant` | AI | 回复文字 / 说"我要调工具" |
| `tool` | 程序 | Day 3 新增——工具执行的结果 |

### 关键代码

```python
from calculator import safe_calc

tools = [{
    "type": "function",
    "function": {
        "name": "calculate",
        "description": "计算数学表达式",
        "parameters": {
            "type": "object",
            "properties": {
                "expression": {
                    "type": "string",
                    "description": "如 '35 * 9/5 + 32'"
                }
            },
            "required": ["expression"]
        }
    }
}]

def chat(user_input: str) -> str:
    messages = [
        {"role": "system", "content": "遇到计算必须用工具"},
        {"role": "user", "content": user_input},
    ]

    # 第一轮：AI 决定要不要调工具
    response = client.chat.completions.create(
        model=os.getenv("OPENAI_MODEL"),
        messages=messages,
        tools=tools,
    )

    msg = response.choices[0].message
    if not msg.tool_calls:
        return msg.content

    # 执行工具
    messages.append(msg)
    for tc in msg.tool_calls:
        args = json.loads(tc.function.arguments)
        result = safe_calc(args["expression"])
        messages.append({
            "role": "tool",
            "tool_call_id": tc.id,
            "content": result,
        })

    # 第二轮：AI 看结果后总结
    final = client.chat.completions.create(
        model=os.getenv("OPENAI_MODEL"),
        messages=messages,
    )
    return final.choices[0].message.content
```

### 记忆口诀

> tools 是说明书，tool_calls 是调令，你执行完塞回 messages，AI 看结果后总结。

## Day 4：插件式工具系统

### 必须记住

| 序号 | 知识点 | 为什么重要 |
|------|--------|-----------|
| 1 | 硬编码 vs 注册表 | 加新工具不改核心代码——这是开闭原则 |
| 2 | `_tools = {}` 全局字典 | 工具名 → 执行函数，运行时动态查找 |
| 3 | `**kwargs` 收集参数 | 函数可以接收任意数量的命名参数 |
| 4 | `**dict` 拆字典 | 把 `{"a": 1, "b": 2}` 拆成 `a=1, b=2` |
| 5 | `dict.get(key)` 安全取值 | key 不存在返回 None，不报错 |

### 核心概念：注册表模式

```
旧方式（硬编码）：                    新方式（注册表）：
┌──────────────────────┐    ┌──────────────────────┐
│ if name == "calculate"│    │ execute("calculate") │
│   → safe_calc(...)   │    │   ↓                  │
│ elif name == "search"│    │ _tools["calculate"]  │
│   → web_search(...)  │    │   → safe_calc        │
│ elif name == "code"  │    │                      │
│   → exec_code(...)   │    │ 不需要改 execute()   │
│                      │    │ 只需要 register()    │
│ 每加一个工具改一次核心 │    │                      │
└──────────────────────┘    └──────────────────────┘
```

### 新语法详解

`*args` 和 `**kwargs` 是 Python 函数参数的特殊写法：

```python
def f(a, b):        # 普通：必须传两个参数
    pass

def f(*args):       # * 收集位置参数为元组
    f(1, 2, 3)      # args = (1, 2, 3)

def f(**kwargs):    # ** 收集命名参数为字典
    f(x=1, y=2)     # kwargs = {"x": 1, "y": 2}
```

`**` 也可以反过来用——拆字典：

```python
args = {"expression": "2+2"}   # AI 返回的参数字典
safe_calc(**args)               # 等价于 safe_calc(expression="2+2")
```

### 工具注册流程

```
1. 定义工具函数：       def safe_calc(expression): ...

2. 注册到全局字典：     register("calculate", safe_calc)
                         → _tools["calculate"] = safe_calc

3. Agent 调用时执行：   execute("calculate", expression="2+2")
                         → 从 _tools 取出 safe_calc
                         → 调用 safe_calc(expression="2+2")
```

### 关键代码

```python
# tools.py —— 工具注册表

_tools = {}  # 全局字典：名字 → 函数

def register(name, func):
    _tools[name] = func          # 注册

def execute(name, **kwargs):
    func = _tools.get(name)      # 查找
    if func is None:
        return f"错误：未知工具 '{name}'"
    try:
        return func(**kwargs)    # 执行（字典拆成参数）
    except Exception as e:
        return f"错误：{e}"

# 启动时注册
register("calculate", safe_calc)
```

### agent.py 改动（只改了两行）

```python
# 改动 1：导入
from tools import execute      # 原来是 from calculator import safe_calc

# 改动 2：执行工具
result = execute(func_name, **func_args)
# 原来是 if func_name == "calculate": result = safe_calc(func_args["expression"])
```

### 记忆口诀

> register 注册，execute 执行，** 拆字典，get 查名字，改工具不碰 agent。

## Day 5：加更多工具——网络搜索 + 代码执行

### 必须记住

| 序号 | 知识点 | 为什么重要 |
|------|--------|-----------|
| 1 | 新工具只需写函数 + register，不改 agent | 注册表模式的好处体现出来了 |
| 2 | `asyncio.create_subprocess_exec` | 子进程隔离——代码崩了不影响主程序 |
| 3 | `asyncio.wait_for` | 超时自动 kill——防止死循环 |
| 4 | `asyncio.iscoroutine()` + `asyncio.run()` | 在同步代码里运行 async 函数 |
| 5 | DuckDuckGo 免费搜索 | 不需要 API Key，pip install 就能用 |

### 核心概念：子进程 vs eval

```
eval()：在主进程里执行代码
  → 代码能访问主进程的所有变量、文件
  → 危险：__import__('os').system('del /')

子进程：在隔离进程里执行代码
  → 子进程崩溃 → 主进程不受影响
  → 子进程死循环 → wait_for 超时自动 kill
  → 子进程删文件 → 用权限控制
```

### 工具注册流程（3 个工具）

```
                ┌─ calculate ─── safe_calc(expression)
                │
ToolRegistry ───┼─ web_search ── web_search(query)
                │
                └─ execute_code ─ execute_code(code)
                
agent.py 只需要: execute(名字, **参数)
谁来都一样——注册表自动分发
```

### register 进化：连 schemas 一起注册

```python
# Day 4：只注册函数

# Day 5：连说明书一起注册

    "calculate",
    "计算数学表达式",           # ← LLM 看的描述

)
```

## Day 1 概念

### Q：`load_dotenv()` 到底干了什么？

打开 `.env` 文件，逐行读取，把每行 `KEY=VALUE` 存到操作系统的环境变量里。

```
.env 文件内容：                   内存里变成：
OPENAI_API_KEY=sk-abc123    →    OPENAI_API_KEY = "sk-abc123"
OPENAI_MODEL=deepseek-chat  →    OPENAI_MODEL = "deepseek-chat"
```

之后 `os.getenv("OPENAI_API_KEY")` 就是从内存取出 `"sk-abc123"`。

**为什么绕这个弯子？** 密码不写死在代码里。代码上传 GitHub，`.env` 不上传。

---

### Q：`messages` 列表为什么每轮都在变？

```
第 1 轮（用户刚发消息）：
  [system规则, user问题]

第 1 轮 LLM 回复后：
  [system规则, user问题, assistant说要调工具]

工具执行后：
  [system规则, user问题, assistant说要调工具, tool工具结果]
  
第 2 轮 LLM 看到完整历史，基于工具结果总结。
```

`messages.append()` 就是往这个历史记录里追加一条。列表越长，AI 能"回忆"的上下文越多。

---

### Q：`response.choices[0].message.content` 为什么要写这么长？

这是 OpenAI 的固定格式，不是 Python 的要求。拆开看每层是什么类型：

```python
response                            # ChatCompletion 对象
  .choices                          # list（候选回复列表，通常只有 1 个）
    [0]                             # 取第一个
      .message                      # ChatCompletionMessage 对象
        .content                    # str（纯文本）
```

如果 DeepSeek 改变了返回格式，比如把 content 放到别的字段，这行代码就废了。所以它叫"API 约定"——你得按人家规定的路径取值。

---

## Day 2 概念

### Q：`eval()` 到底危险在哪？

```python
eval("2 + 2")                         # 安全，返回 4
eval("__import__('os').system('dir')") # 危险！会真的列出你电脑的文件
eval("open('secret.txt').read()")     # 危险！会真的读你电脑上的文件
```

`eval()` 不区分"数学算式"和"系统命令"——它什么 Python 代码都执行。所以需要沙箱。

---

### Q：`compile(str, "", "eval")` 是什么？

把字符串**翻译**成字节码，但**不执行**。

```python
code = compile("2 + 2", "", "eval")    # 翻译成功，返回 code 对象
code = compile("import os", "", "eval") # 翻译失败，抛出 SyntaxError
```

"eval" 模式只接受**表达式**（有返回值的），不接受**语句**（import、print、for、def 等）。这是第一道防线。

---

### Q：`code.co_names` 是什么？

编译后的字节码对象有一个属性 `co_names`，它记录了代码里**提到的所有变量名**。

```python
code = compile("sqrt(4) + pi", "", "eval")
code.co_names  # → ("sqrt", "pi")        # 只提到了这两个

code = compile("__import__('os')", "", "eval")
code.co_names  # → ("__import__",)        # 提到了危险函数！
```

检查 `co_names` 里每个名字是否在白名单里——这就是第三道防线。

---

### Q：`{"__builtins__": {}}` 是什么意思？

Python 执行代码时，有一个叫 `__builtins__` 的字典存着所有内置函数名（`open`、`print`、`__import__`、`exec`...）。把它设为空字典 `{}`，这些函数就全没了。

```python
eval(code, {"__builtins__": {}}, allowed)
#                ↑
#          清空所有内置函数，只给 allowed 里的数学函数
```

---

## Day 3 概念

### Q：`msg.tool_calls` 是什么？

LLM 的回复有两种情况：

```python
# 情况 1：直接回答 → content 有值，tool_calls 是 None
msg.content = "1+1=2"
msg.tool_calls = None

# 情况 2：要求调工具 → tool_calls 有值
msg.tool_calls = [
    {
        "id": "call_xxx",
        "function": {
            "name": "calculate",
            "arguments": '{"expression": "35*9/5+32"}'
        }
    }
]
```

`if not msg.tool_calls` 就是判断"AI 有没有要求调工具"。

---

### Q：`json.loads()` 为什么必须用？

AI 返回的 `tc.function.arguments` 是**字符串**，不是字典：

```python
# 这是字符串（不能直接 ["expression"] 取值）
'{"expression": "35*9/5+32"}'

# json.loads() 之后变成字典（可以 ["expression"] 取值）
{"expression": "35*9/5+32"}
```

不用 `json.loads()`，`func_args["expression"]` 会报错——你不能对字符串用方括号取值。

---

### Q：为什么需要两轮调用 LLM？

第一轮：AI 看到用户问题 + 工具列表 → 决定"需要调 calculate" → 返回 tool_calls

但你执行完工具后，AI 还不知道结果。你得把结果塞进 messages，再调一次：

第二轮：AI 看到完整对话（含工具结果） → "35*9/5+32=95，所以答案是 95°F"

如果 AI 需要调多个工具（比如先搜索再计算），就是三轮、四轮——这就是 ReAct 循环。

---

## Day 4 概念

### Q：`**kwargs` 到底做了什么？

`**` 有两种用法：

**用法一：收集参数（函数定义时）**

```python
def f(**kwargs):
    print(kwargs)

f(a=1, b=2, c=3)  # kwargs = {"a": 1, "b": 2, "c": 3}
```

**用法二：拆开字典（函数调用时）**

```python
args = {"expression": "2+2"}
safe_calc(**args)  # 等价于 safe_calc(expression="2+2")
```

你的 `execute` 函数用 `**kwargs` 接收任意参数，再用 `func(**kwargs)` 原封不动传给目标函数。这样不管工具需要什么参数（expression、query、code...），`execute` 都不需要知道。

---

### Q：`dict.get(key)` 和 `dict[key]` 有什么区别？

```python
_tools = {"calculate": safe_calc}

_tools["calculate"]   # → safe_calc 函数
_tools["unknown"]     # → KeyError! 程序崩溃

_tools.get("calculate")  # → safe_calc 函数
_tools.get("unknown")    # → None（不报错）
```

用 `.get()` 安全——key 不存在返回 None，你可以自己处理。

### Q：注册表模式的本质是什么？

```
没有注册表：                       有注册表：
                                  
if name == "A":  调函数A            _tools = {"A": 函数A, "B": 函数B}
elif name == "B": 调函数B            execute(name) → _tools[name]()
elif name == "C": 调函数C           
                                  加工具 C：register("C", 函数C)
加工具 C：改核心代码，加 elif         核心代码不动
```

本质：**用字典代替 if-else**。字典可以运行时增删，if-else 是写死的。

## Day 5 概念

### Q：`async/await` 是什么？为什么要用？

普通函数：从头执行到尾，中间不能停。

```python
def f():
    a = 1          # 执行
    sleep(5)       # 等 5 秒，CPU 闲着
    b = 2          # 5 秒后才执行
```

异步函数：中间可以"暂停"去做别的事。

```python
async def f():
    a = 1               # 执行
    await sleep(5)      # 暂停！CPU 去处理其他请求
    b = 2               # 5 秒后回来继续
```

子进程执行可能需要 5-10 秒。如果用普通函数，整个服务器卡住等它。用 `async/await`，等的时候 CPU 去处理其他用户的消息。

**`await` = "这里要等，我先去做别的事，好了叫我"。**

---

### Q：`asyncio.run()` 又是干嘛的？

`async` 函数不能直接调用：

```python
result = execute_code(code)  # 返回 coroutine 对象，不是实际结果！

result = asyncio.run(execute_code(code))  # 正确：跑完异步函数，拿到结果
```

`asyncio.run()` = "启动事件循环，等这个异步函数跑完，把结果给我"。

---

### Q：子进程隔离到底隔离了什么？

```
主进程（你的 server.py）          子进程（跑用户代码）
├── 内存空间 A                    ├── 内存空间 B（完全独立）
├── 变量、函数、数据               ├── 自己的变量
├── 文件句柄                      ├── 自己的文件句柄
└── 网络连接                      └── 自己的网络连接

子进程崩溃 → 内存 B 释放 → 主进程毫发无损
子进程死循环 → wait_for 超时 → kill → 主进程继续
```

## Day 6 概念

### Q：`@app.get("/")` 那个 `@` 是什么？

这叫**装饰器**（decorator）。写成：

```python
@app.get("/")
async def home():
    return "首页"
```

本质等价于：

```python
def home():
    return "首页"
home = app.get("/")(home)  # 把函数注册到路径 "/"
```

装饰器的作用：**"把这个函数绑定到 `/` 这个路径上"**。用户访问 `http://localhost:8000/` → FastAPI 调用 `home()`。

---

### Q：`yield` 和 `return` 有什么区别？

```python
def f():
    return 1    # 返回 1，函数结束，后面代码不再执行
    return 2    # 永远不会执行

def g():
    yield 1     # 返回 1，但函数暂停（没结束）
    yield 2     # 下次调用继续从这执行，返回 2
    yield 3     # 再来一次，返回 3
```

`return` = 一次性全返回，函数结束
`yield` = 一条一条返回，函数保持"待机"状态

SSE 流式推送的核心就是 `yield`——每产生一条消息就推给前端，而不是等所有消息都生成完再一次性发。

---

### Q：SSE 的 `data:` 格式为什么长那样？

SSE 协议规定每条消息格式必须是：

```
data: {json内容}\n\n
```

- `data:` 前缀 → 浏览器 EventSource API 认得
- `\n\n` 双换行 → 消息分隔符

你的 `server.py` 里所有 `yield "data: ..."` 都在遵守这个协议。前端 JS 的 `EventSource` 对象收到这些 `data:` 行，自动解析成事件。
