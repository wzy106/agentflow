## Day 4：HTTP / REST

### 为什么 agent 需要懂 HTTP

Agent 调用外部工具（查天气、发邮件、搜网页）本质上都是发 HTTP 请求。你不理解 HTTP，就看不懂工具调用到底发生了什么。

### 必须记住

| 序号 | 知识点 | 为什么重要 |
|------|--------|-----------|
| 1 | HTTP 就是"请求-响应"四个字 | 所有 web 通信的基础范式 |
| 2 | GET 读数据，POST 写数据 | 调用 API 时写错方法直接报 405 |
| 3 | URL 三要素：协议 + 域名 + 路径 | 看懂 API 文档的前提 |
| 4 | JSON 是 HTTP 的通用语言 | agent 和工具之间传数据全靠它 |
| 5 | 状态码 200/400/404/500 | 调试时看状态码就知道哪里错了 |

### 核心概念：一次 HTTP 请求长什么样

```
你（客户端）                                      服务器
    │                                                │
    │  ──── GET /api/weather?city=fuzhou ────▶       │
    │         Host: api.weather.com                   │
    │         Accept: application/json                │
    │                                                │
    │  ◀──── 200 OK ──────────────────────────       │
    │         Content-Type: application/json          │
    │                                                │
    │         {"temp": 25, "status": "晴"}            │
    │                                                │
```

**拆解上面的请求**：

```
GET /api/weather?city=fuzhou HTTP/1.1       ← 请求行：方法 + 路径 + 版本
Host: api.weather.com                        ← 请求头（Headers）：元信息
Accept: application/json                     ← "我只要 JSON"

                    ← 空行分隔

(没有请求体，GET 不带 body)                   ← 请求体（Body）：数据载荷
```

**拆解上面的响应**：

```
HTTP/1.1 200 OK                              ← 状态行：200=成功
Content-Type: application/json               ← 响应头："下面是 JSON"

                    ← 空行分隔

{"temp": 25, "status": "晴"}                 ← 响应体：真正有用的数据
```

### 四种请求方法（Agent 用前两种就够）

| 方法 | 含义 | 比喻 | 什么时候用 |
|------|------|------|-----------|
| **GET** | 读取 | 看书 | 查天气、搜信息 |
| **POST** | 创建 | 写新页 | 发邮件、调工具计算 |
| PUT | 整体替换 | 撕掉重写 | 更新配置 |
| DELETE | 删除 | 撕掉 | 删文件 |

### URL 结构拆解

```
https://api.example.com:8080/users/123?page=1&size=10
└─┬─┘ └──────┬──────┘ └┬┘ └───┬────┘ └──────┬──────┘
 协议      域名        端口   路径          查询参数

协议：    https = 加密的 http（现在全用 https）
域名：    api.example.com → DNS 解析成 IP 地址
端口：    8080（默认 80 for http, 443 for https，默认可省略）
路径：    /users/123 → 服务器内部路由到这里
查询参数：?page=1&size=10 → 附加筛选条件
```

### REST 是什么

REST 不是协议，是一种**设计风格**——用 URL 表示资源，用 HTTP 方法表示操作。

```
传统方式（乱）：              REST 风格（清晰）：
GET  /getUserById?id=5        GET    /users/5          ← 查用户5
POST /deleteUser               DELETE /users/5          ← 删用户5
POST /createUser               POST   /users            ← 新建用户
POST /updateUser               PUT    /users/5          ← 更新用户5
```

**一句话总结**：URL 只表示"什么东西"，HTTP 方法表示"干什么"。

### JSON：agent 和工具之间的通用语言

```json
{
    "name": "calculate",
    "arguments": {
        "expression": "35 * 9/5 + 32"
    }
}
```

所有现代 API 都用 JSON。Python 里 `json.loads()` 把字符串变成字典，`json.dumps()` 把字典变成字符串。

### Python 发 HTTP 请求

```python
import requests  # pip install requests

# GET 请求：查天气
resp = requests.get(
    "https://api.weather.com/v1/current",
    params={"city": "fuzhou"},              # 自动拼成 ?city=fuzhou
    headers={"Authorization": "Bearer xxx"} # 认证令牌
)
print(resp.status_code)  # 200 表示成功
print(resp.json())       # 直接把 JSON 响应转成字典

# POST 请求：发数据
resp = requests.post(
    "http://localhost:8000/calculate",
    json={"expression": "2 + 3"},  # 自动转 JSON 放 body 里
)
print(resp.json())  # {"result": 5}
```

### 常见状态码速查

| 状态码 | 含义 | 你该怎么做 |
|:---:|------|-----------|
| 200 | 成功 | 正常处理数据 |
| 301/302 | 重定向 | `requests` 默认自动跟随 |
| 400 | 你发错了 | 检查请求参数格式 |
| 401 | 没认证 | 检查 API Key 是否传了 |
| 403 | 没权限 | 检查 API Key 是否有效 |
| 404 | 不存在 | 检查 URL 路径对不对 |
| 422 | 参数不对 | 检查请求体字段名和类型 |
| 429 | 请求太多 | `time.sleep()` 等一下再试 |
| 500 | 服务器炸了 | 重试，不行就看对方文档 |

### 记忆口诀

> GET 是看，POST 是写，URL 找资源，方法定操作，JSON 传数据，状态码看结果。

## Day 5：FastAPI 搭服务

### 为什么需要 FastAPI

你的 `agent.py` 目前只能在终端里跑。把它包一层 FastAPI，就变成一个 **web 服务**，有 URL 地址，别人（或前端页面）通过 HTTP 就能调用它。这就是"Agent as a Service"。

### 必须记住

| 序号 | 知识点 | 为什么重要 |
|------|--------|-----------|
| 1 | `@app.get("/path")` 装饰器定义路由 | FastAPI 最核心语法 |
| 2 | Pydantic 模型定义请求/响应格式 | 自动校验 + 自动生成文档 |
| 3 | `uvicorn` 是 ASGI 服务器 | 把 FastAPI 跑起来的东西 |
| 4 | 路径参数 vs 查询参数 | 设计 API 的基本功 |
| 5 | Swagger 文档自动生成 | 调试神器，不用 Postman |

### 从零到第一个服务（15 分钟上手）

**安装**：
```bash
pip install fastapi uvicorn
```

**第一个服务** — 创建 `server.py`：

```python
from fastapi import FastAPI

# 创建应用（整个服务的入口）
app = FastAPI(title="我的 Agent 服务")


# ① 最简单的 GET 接口
@app.get("/")
def root():
    """访问 http://localhost:8000/ 时触发"""
    return {"message": "Agent 服务运行中"}


# ② 路径参数：/users/5 → user_id = 5
@app.get("/users/{user_id}")
def get_user(user_id: int):
    """FastAPI 自动把字符串 '5' 转成整数 5"""
    return {"user_id": user_id, "name": f"用户{user_id}"}


# ③ 查询参数：/search?keyword=python&limit=10
@app.get("/search")
def search(keyword: str, limit: int = 10):
    """limit 有默认值 10，所以是可选的"""
    return {"keyword": keyword, "limit": limit}


# ④ POST 接口：接收 JSON 请求体
from pydantic import BaseModel

class CalcRequest(BaseModel):   # Pydantic 模型 = 数据格式说明书
    expression: str             # 必填字段
    precision: int = 2          # 可选字段，默认2

class CalcResponse(BaseModel):
    expression: str
    result: float

@app.post("/calculate")
def calculate(req: CalcRequest):
    """FastAPI 自动校验：缺少 expression 直接返回 422"""
    # 这里换成你的 safe_calc
    result = eval(req.expression, {"__builtins__": {}}, {})
    return CalcResponse(
        expression=req.expression,
        result=round(result, req.precision)
    )
```

**启动服务**：
```bash
uvicorn server:app --reload
#        │        │    └── 代码改了自动重启（开发用）
#        │        └── 应用实例名
#        └── 文件名（server.py）
```

### 把 agent.py 变成 API 服务

```python
# agent_server.py —— 把 agent 暴露成 API
from fastapi import FastAPI
from pydantic import BaseModel
from agent import chat  # 直接导入你 Day 3 写的 chat 函数

app = FastAPI(title="Agent API")

class ChatRequest(BaseModel):
    message: str

class ChatResponse(BaseModel):
    reply: str

@app.post("/chat", response_model=ChatResponse)
def agent_chat(req: ChatRequest):
    reply = chat(req.message)
    return ChatResponse(reply=reply)
```

### 中间件（拦截所有请求）

```python
from fastapi import Request
import time

@app.middleware("http")
async def log_requests(request: Request, call_next):
    """记录每个请求的耗时"""
    start = time.time()
    response = await call_next(request)  # 放行给下一个处理
    duration = time.time() - start
    print(f"{request.method} {request.url.path} → {duration:.3f}s")
    return response
```

agent 场景里，中间件常用于：打印日志、验证 API Key、限流计数。

## Day 6：数据库 SQL + Redis

### 为什么 agent 需要数据库

Agent 现在没有记忆——每次对话都是新的。数据库解决两个问题：
- **SQL（持久化）**：用户信息、对话历史、工具调用记录，关了重开还在
- **Redis（缓存）**：临时状态、限流计数、短期记忆，关了消失但极快

### 6.1 SQL（以 SQLite 入门）

### 必须记住

| 序号 | 知识点 | 为什么重要 |
|------|--------|-----------|
| 1 | CRUD = 增删改查(Create, Read, Update, Delete) | 所有数据库操作的四个起点 |
| 2 | SQLite 零配置，一个文件就是数据库 | 开发阶段完美选择 |
| 3 | `cursor.execute(sql, params)` 防注入 | 永远不要拼字符串 |
| 4 | 主键 PRIMARY KEY 唯一标识一行 | 关联数据的基础 |
| 5 | `connection.commit()` 才真正写入 | 忘了这句数据白插了 |

### 核心概念：SQL 四句话

你需要背的就四类 SQL：

```sql
-- ① 创建表（规划好"这张表存什么字段"）
CREATE TABLE conversations (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,  -- 自增主键
    user_id   TEXT NOT NULL,                       -- 谁说的
    role      TEXT NOT NULL,                       -- user/assistant/tool
    content   TEXT NOT NULL,                       -- 说了什么
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP  -- 时间戳
);

-- ② 插入数据（INSERT）
INSERT INTO conversations (user_id, role, content)
VALUES ('alice', 'user', '今天天气怎么样');

-- ③ 查询数据（SELECT）
SELECT role, content FROM conversations
WHERE user_id = 'alice'
ORDER BY created_at DESC    -- 最新的在前
LIMIT 10;                   -- 只拿最近 10 条

-- ④ 更新和删除（UPDATE / DELETE）
UPDATE conversations SET content = '修改后' WHERE id = 1;
DELETE FROM conversations WHERE id = 1;
```

### Python 操作 SQLite

```python
import sqlite3

# 连接数据库（文件不存在会自动创建）
conn = sqlite3.connect("agent.db")
cursor = conn.cursor()

# 建表
cursor.execute("""
    CREATE TABLE IF NOT EXISTS conversations (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        user_id TEXT NOT NULL,
        role TEXT NOT NULL,
        content TEXT NOT NULL,
        created_at DATETIME DEFAULT CURRENT_TIMESTAMP
    )
""")
conn.commit()

# 插入（用 ? 占位符防 SQL 注入）
cursor.execute(
    "INSERT INTO conversations (user_id, role, content) VALUES (?, ?, ?)",
    ("alice", "user", "你好")  # 第二个参数是元组
)
conn.commit()
print(f"插入成功，id={cursor.lastrowid}")  # 拿到自增 id

# 查询
cursor.execute(
    "SELECT role, content, created_at FROM conversations WHERE user_id = ? ORDER BY id DESC LIMIT 10",
    ("alice",)
)
rows = cursor.fetchall()  # 拿到所有结果
for role, content, timestamp in rows:
    print(f"[{timestamp}] {role}: {content}")

# 用完关闭
conn.close()
```

### 给 agent 加上对话记忆（改造 agent.py）

```python
# agent_with_memory.py
import sqlite3
from datetime import datetime

class MemoryStore:
    """Agent 的长期记忆"""
    
    def __init__(self, db_path="agent.db"):
        self.conn = sqlite3.connect(db_path)
        self.conn.execute("""
            CREATE TABLE IF NOT EXISTS conversations (
                id INTEGER PRIMARY KEY AUTOINCREMENT,
                session_id TEXT NOT NULL,
                role TEXT NOT NULL,
                content TEXT NOT NULL,
                created_at DATETIME DEFAULT CURRENT_TIMESTAMP
            )
        """)
        self.conn.commit()
    
    def save(self, session_id: str, role: str, content: str):
        self.conn.execute(
            "INSERT INTO conversations (session_id, role, content) VALUES (?, ?, ?)",
            (session_id, role, content)
        )
        self.conn.commit()
    
    def get_history(self, session_id: str, limit: int = 10):
        rows = self.conn.execute(
            "SELECT role, content FROM conversations WHERE session_id = ? ORDER BY id DESC LIMIT ?",
            (session_id, limit)
        ).fetchall()
        # 返回 messages 格式（倒回来恢复时间顺序）
        return [{"role": r, "content": c} for r, c in reversed(rows)]
```

### 6.2 Redis（缓存/限流/临时状态）

**为什么要学 Redis**：
- 读一次 SQL 耗时 1-10ms，读 Redis 耗时 <1ms
- agent 反复查同一个数据（如系统配置、用户权限），用 Redis 缓存
- agent 调 API 要限流（每分钟最多 10 次），用 Redis 计数器

### 必须记住

| 序号 | 知识点 | 为什么重要 |
|------|--------|-----------|
| 1 | Redis 是内存数据库，`SET key value` | 重启就没了，适合临时数据 |
| 2 | `EXPIRE key 60` 设置 60 秒过期 | 自动过期不用手动删 |
| 3 | `INCR key` 原子自增 | 限流计数的基础操作 |
| 4 | Redis 数据结构：string/list/hash/set | string 最常用，其他按需学 |
| 5 | 重启 Redis 数据全丢（除非开启持久化） | 重要数据存 SQL，别存 Redis |

### Python 操作 Redis

```bash
pip install redis
```

```python
import redis

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

# 基本操作：存值和取值
r.set("user:alice:last_login", "2026-05-29")
print(r.get("user:alice:last_login"))  # "2026-05-29"

# 设置过期时间（60 秒后自动删除）
r.setex("session:token123", 60, "alice")  # 60 秒后过期

# 自增（限流计数器）
count = r.incr("rate_limit:alice")  # 第一次返回 1，第二次 2...
if count == 1:
    r.expire("rate_limit:alice", 60)  # 第一次才设过期
if count > 10:
    print("请求太频繁，请稍后")

# 存储 JSON（序列化后存）
import json
history = [{"role": "user", "content": "你好"}]
r.setex("chat:session_001", 3600, json.dumps(history))  # 1小时过期
```

### 实际应用：Agent 的限流和缓存

```python
# agent_utils.py —— Agent 实际会用到的数据库工具
import redis
import json
from datetime import datetime

r = redis.Redis(host='localhost', port=6379, decode_responses=True)

def check_rate_limit(user_id: str, max_per_minute: int = 10) -> bool:
    """检查用户是否超过调用频率限制"""
    key = f"rate:{user_id}"
    count = r.incr(key)
    if count == 1:
        r.expire(key, 60)
    return count <= max_per_minute

def cache_api_result(key: str, data: dict, ttl: int = 300):
    """缓存 API 调用结果，5 分钟过期"""
    r.setex(f"cache:{key}", ttl, json.dumps(data))

def get_cached_result(key: str) -> dict | None:
    """读缓存，没有返回 None"""
    raw = r.get(f"cache:{key}")
    return json.loads(raw) if raw else None

# 使用示例
def smart_weather_query(city: str) -> dict:
    """聪明查天气：先看缓存，没有再调 API"""
    cached = get_cached_result(f"weather:{city}")
    if cached:
        return cached  # 缓存命中，不用调 API
    
    # 缓存没命中，调 API
    result = call_weather_api(city)  # 假设这是发 HTTP 请求
    cache_api_result(f"weather:{city}", result, ttl=600)  # 缓存 10 分钟
    return result
```

### SQL vs Redis 选择表

| 场景 | 用什么 | 为什么 |
|:---|:---:|------|
| 用户账号信息 | SQL | 必须持久化，丢了就没了 |
| 对话历史 | SQL | 需要长期保存 |
| 当前会话上下文 | Redis | 会话结束就删 |
| API 调用限流计数 | Redis | 重启无所谓，原子自增 |
| 天气预报缓存 | Redis | 过期自动删，不用维护 |
| 全局配置 | Redis + SQL | Redis 读缓存，SQL 持久化 |

### 记忆口诀

> SQL 存永久，Redis 做缓存；set 赋值 get 取值，INCR 计数器，EXPIRE 自动删。

## Day 7：异步编程 asyncio

### 为什么 agent 需要异步

假设你的 agent 需要同时调三个工具：搜索网页、查数据库、调计算器。同步代码是一个接一个等：3s + 2s + 1s = 6 秒。异步代码三个同时发，取最慢的那个：3 秒。

```
同步（串行）：              异步（并行）：
搜索 ████████ (3s)          搜索 ████████ (3s)
                ↓           数据库 ████ (2s)
数据库 ████ (2s)            计算器 ██ (1s)
                ↓              ↓ 三件事同时启动
计算器 ██ (1s)              3 秒后全部完成
                ↓
总共 6 秒                   总共 3 秒
```

### 必须记住

| 序号 | 知识点 | 为什么重要 |
|------|--------|-----------|
| 1 | `async def` + `await` 两个关键字 | Python 异步的全部语法 |
| 2 | `asyncio.gather()` 并行跑多个任务 | agent 同时调多个工具的核心 |
| 3 | 异步 ≠ 多线程（单线程 + 事件循环） | 不涉及线程安全问题 |
| 4 | `await` 只能用在 `async def` 函数里 | 忘写 `await` 是最常见的 bug |
| 5 | 异步 IO 框架和同步代码不混用 | `requests` 同步，`httpx` 异步 |

### 核心概念：async/await 到底干了什么

```python
import asyncio

# 同步版本：每个 sleep 阻塞整个线程
def sync_main():
    print("开始")
    time.sleep(1)  # 线程在这里卡住 1 秒，啥也干不了
    print("结束")

# 异步版本：sleep 时切换去做别的事
async def async_main():
    print("开始")
    await asyncio.sleep(1)  # 在这里暂停，让事件循环去跑别的任务
    print("结束")
```

**关键理解**：`await` 的意思是"这件事需要等，我先去干别的，好了叫我"。CPU 没有闲着，而是去执行其他异步任务了。

## Day 8：JWT 认证完整实现

### 为什么必须学

你的 agent 服务如果没有任何认证，任何人知道 URL 就能调用。JWT 是让"只有登录过的人才能用 agent"的最轻量方案。

### 核心概念：JWT 到底长什么样

JWT 就是用 `.` 把三段 base64 串起来的字符串。你复制下面这串到 [jwt.io](https://jwt.io) 就能看到内容：

```
eyJhbGciOiJIUzI1NiJ9.eyJ1c2VyX2lkIjoiYWxpY2UiLCJleHAiOjE3MDAwMDAwMDB9.签名部分
  └── Header ──┘  └──────── Payload ──────────┘  └─ Signature ─┘
  算法=HS256        数据：user_id + 过期时间           防篡改签名
```

**三个关键认知**：
1. Payload 是 base64 编码，**不是加密**——任何人解码能看到内容，所以**不要放密码**
2. Signature 的作用是**防篡改**——改了 Payload，签名就验不过
3. 服务器不存 JWT（无状态），靠签名验证真伪

### 必须记住

| 序号 | 知识点 | 为什么重要 |
|------|--------|-----------|
| 1 | JWT 三段结构：Header.Payload.Signature | 面试必问，看懂 jwt.io |
| 2 | `jwt.encode(payload, secret)` 签发 | 登录时生成 token |
| 3 | `jwt.decode(token, secret)` 验证 | 每次请求时验证 |
| 4 | `Depends(get_current_user)` 依赖注入 | FastAPI 最优雅的认证方式 |
| 5 | 密码必须 hash（`bcrypt`/`hashlib`） | 明文存密码 = 等着出安全事故 |

### 完整实现：从注册到受保护接口

**安装**：

**第一步：auth 工具库** — 创建 `auth_utils.py`：

```python
import jwt
import bcrypt
from datetime import datetime, timedelta, timezone

SECRET_KEY = "你的密钥-生产环境用复杂随机字符串"  # 实际放 .env 里
ALGORITHM = "HS256"
TOKEN_EXPIRE_MINUTES = 60  # token 1小时过期


# ====== 密码处理 ======

def hash_password(password: str) -> str:
    """把明文密码变成哈希值存入数据库（永远不要存明文）"""
    salt = bcrypt.gensalt()
    return bcrypt.hashpw(password.encode('utf-8'), salt).decode('utf-8')

def verify_password(password: str, hashed: str) -> bool:
    """验证密码是否匹配"""
    return bcrypt.checkpw(password.encode('utf-8'), hashed.encode('utf-8'))


# ====== Token 处理 ======

def create_access_token(user_id: str) -> str:
    """登录时调用——生成 JWT"""
    payload = {
        "user_id": user_id,
        "exp": datetime.now(timezone.utc) + timedelta(minutes=TOKEN_EXPIRE_MINUTES),
        "iat": datetime.now(timezone.utc),  # 签发时间
    }
    return jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)

def decode_token(token: str) -> dict:
    """验证 token 并返回里面的数据。不合法直接抛异常"""
    return jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
    # 如果过期或签名不对，自动抛出 jwt.ExpiredSignatureError 或 jwt.InvalidTokenError
```

**第二步：FastAPI 依赖注入** — 追加到 `auth_utils.py`：

```python
from fastapi import Depends, HTTPException, status
from fastapi.security import HTTPBearer, HTTPAuthorizationCredentials

security = HTTPBearer()  # 告诉 FastAPI 从 Header 取 Authorization: Bearer <token>


async def get_current_user(
    credentials: HTTPAuthorizationCredentials = Depends(security)
) -> str:
    """
    在任何一个需要认证的接口里加上这个 Depends，FastAPI 自动：
    1. 从请求头取 Authorization: Bearer <token>
    2. 验证 token 有效
    3. 返回 user_id
    """
    token = credentials.credentials  # 拿到 "Bearer xxx" 后面的 token 部分
    try:
        payload = decode_token(token)
        return payload["user_id"]
    except jwt.ExpiredTokenError:
        raise HTTPException(status_code=401, detail="Token 已过期，请重新登录")
    except jwt.InvalidTokenError:
        raise HTTPException(status_code=401, detail="Token 无效")
```

**第三步：把认证接入 FastAPI** — 创建 `auth_server.py`：

```python
from fastapi import FastAPI, Depends, HTTPException
from pydantic import BaseModel
import sqlite3
from auth_utils import hash_password, verify_password, create_access_token, get_current_user

app = FastAPI(title="Agent 认证服务")


# ====== 模拟用户数据库 ======
def init_db():
    conn = sqlite3.connect("users.db")
    conn.execute("""
        CREATE TABLE IF NOT EXISTS users (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            username TEXT UNIQUE NOT NULL,
            password_hash TEXT NOT NULL,
            created_at DATETIME DEFAULT CURRENT_TIMESTAMP
        )
    """)
    conn.commit()
    return conn

init_db()


# ====== Pydantic 请求模型 ======
class RegisterRequest(BaseModel):
    username: str
    password: str

class LoginRequest(BaseModel):
    username: str
    password: str

class TokenResponse(BaseModel):
    access_token: str
    token_type: str = "bearer"


# ====== 注册接口 ======
@app.post("/register")
def register(req: RegisterRequest):
    conn = sqlite3.connect("users.db")
    existing = conn.execute(
        "SELECT id FROM users WHERE username = ?", (req.username,)
    ).fetchone()
    if existing:
        conn.close()
        raise HTTPException(status_code=400, detail="用户名已存在")

    hashed = hash_password(req.password)
    conn.execute(
        "INSERT INTO users (username, password_hash) VALUES (?, ?)",
        (req.username, hashed)
    )
    conn.commit()
    conn.close()
    return {"message": f"用户 {req.username} 注册成功"}


# ====== 登录接口 ======
@app.post("/login", response_model=TokenResponse)
def login(req: LoginRequest):
    conn = sqlite3.connect("users.db")
    row = conn.execute(
        "SELECT password_hash FROM users WHERE username = ?", (req.username,)
    ).fetchone()
    conn.close()

    if not row or not verify_password(req.password, row[0]):
        raise HTTPException(status_code=401, detail="用户名或密码错误")

    token = create_access_token(req.username)
    return TokenResponse(access_token=token)


# ====== 受保护的接口 ======
@app.get("/me")
def who_am_i(user_id: str = Depends(get_current_user)):
    """需要登录才能访问——传入 Depends(get_current_user)"""
    return {"message": f"你好 {user_id}，你已经通过认证"}


@app.post("/agent/chat")
def agent_chat(message: str, user_id: str = Depends(get_current_user)):
    """Agent 接口——只有登录用户才能调用"""
    # 这里调你的 agent
    return {"user": user_id, "reply": f"Agent 收到了你的消息：{message}"}
```

### 整个流程跑一遍

```
1. POST /register  {"username":"alice","password":"123456"}
   → {"message": "用户 alice 注册成功"}
   (数据库里存的是哈希值 "$2b$12$...", 不是 "123456")

2. POST /login  {"username":"alice","password":"123456"}
   → {"access_token": "eyJhbG...", "token_type": "bearer"}

3. GET /me  (Header: Authorization: Bearer eyJhbG...)
   → {"message": "你好 alice，你已经通过认证"}

4. GET /me  (不带 Header)
   → 401 {"detail": "Not authenticated"}

5. GET /me  (带过期的 token)
   → 401 {"detail": "Token 已过期，请重新登录"}
```

### Token 存储策略：前端存哪，后端怎么用

```
后端（你管）：
  - 不存 token（无状态）
  - 每次请求从 Header 取 → decode → 验签名和过期时间
  - 想"登出"某用户 → 在 Redis 里加黑名单（简单方案）

前端（不是你写的，但要知道）：
  - Token 存在 localStorage 或 cookie
  - 每次请求在 Header 加 Authorization: Bearer <token>
  - Token 快过期时调用 /refresh 刷新（可选）
```

### 记忆口诀

> 注册时 hash 存密码，登录时比对发 token，请求时 Header 带 token，FastAPI Depends 自动验。

### 9.1 Python 日志系统

**为什么不能只用 print**：
- print 只在控制台显示，关了终端就没了
- 出 bug 时需要看几天前的日志，print 做不到
- 生产环境的日志需要分级（DEBUG/INFO/WARNING/ERROR）

**最简可用配置** — 直接复制到 `logger.py`：

```python
import logging
import sys
from pathlib import Path

def setup_logger(name: str = "agent") -> logging.Logger:
    """配置一个同时输出到控制台和文件的日志器"""

    logger = logging.getLogger(name)
    logger.setLevel(logging.DEBUG)  # DEBUG 以上全记录

    # 格式：2026-05-29 14:30:01 [INFO] agent_server.py:42 用户 alice 调用了 /chat
    formatter = logging.Formatter(
        "%(asctime)s [%(levelname)s] %(filename)s:%(lineno)d — %(message)s",
        datefmt="%Y-%m-%d %H:%M:%S",
    )

    # ① 输出到控制台（INFO 以上才显示，避免刷屏）
    console_handler = logging.StreamHandler(sys.stdout)
    console_handler.setLevel(logging.INFO)
    console_handler.setFormatter(formatter)
    logger.addHandler(console_handler)

    # ② 输出到文件（DEBUG 以上全存，出 bug 时查）
    log_dir = Path("logs")
    log_dir.mkdir(exist_ok=True)
    file_handler = logging.FileHandler(log_dir / "agent.log", encoding="utf-8")
    file_handler.setLevel(logging.DEBUG)
    file_handler.setFormatter(formatter)
    logger.addHandler(file_handler)

    return logger


# 使用示例
logger = setup_logger()

@app.post("/chat")
def chat(req: ChatRequest, user_id: str = Depends(get_current_user)):
    logger.info(f"用户 {user_id} 发送消息：{req.message[:50]}...")
    try:
        reply = agent_chat(req.message)
        logger.info(f"回复成功，长度 {len(reply)}")
        return {"reply": reply}
    except Exception as e:
        logger.error(f"Agent 调用失败：{e}", exc_info=True)  # exc_info=True 记录完整堆栈
        raise HTTPException(status_code=500, detail="服务内部错误")
```

**日志级别怎么选**：

| 级别 | 含义 | 什么时候用 |
|:---:|------|------|
| DEBUG | 调试信息 | 开发时打印变量值，生产关掉 |
| INFO | 正常事件 | 谁调了什么接口、耗时多少 |
| WARNING | 异常但能继续 | Token 快过期、API 调用重试了 |
| ERROR | 错误需处理 | 工具调用失败、数据库连接断开 |
| CRITICAL | 服务要挂了 | 内存不足、磁盘满 |

main ────●────●────●────●  (稳定版本，随时能部署)

**场景 3：解决冲突**

>>>>>>> main

### 10.2 pytest 测试

**为什么必须写测试**：
- 后端接口有几十个，你改了一个，怎么知道其他没被搞坏？→ 跑测试
- 面试官问"你怎么保证代码质量"→ "每个接口都有对应测试"
- 大厂开发流程：提交代码 → CI 自动跑测试 → 不通过合并不了

**安装**：
```bash
pip install pytest httpx
```

**测试 FastAPI 接口** — 创建 `test_server.py`：

```python
import pytest
from fastapi.testclient import TestClient
from agent_server import app  # 你的 FastAPI 应用

client = TestClient(app)  # 不需要真的启动服务器


# ====== 测试注册 ======
def test_register_success():
    """测试正常注册"""
    response = client.post("/register", json={
        "username": "testuser",
        "password": "test123"
    })
    assert response.status_code == 200
    assert "注册成功" in response.json()["message"]


def test_register_duplicate():
    """测试重复注册"""
    client.post("/register", json={"username": "dup", "password": "123"})
    response = client.post("/register", json={"username": "dup", "password": "123"})
    assert response.status_code == 400  # 应该拒绝


# ====== 测试登录 ======
def test_login_success():
    """正常登录拿到 token"""
    response = client.post("/login", json={
        "username": "testuser",
        "password": "test123"
    })
    assert response.status_code == 200
    data = response.json()
    assert "access_token" in data
    assert data["token_type"] == "bearer"


def test_login_wrong_password():
    """错误密码被拒绝"""
    response = client.post("/login", json={
        "username": "testuser",
        "password": "wrong_password"
    })
    assert response.status_code == 401


# ====== 测试受保护的接口 ======
def test_me_without_token():
    """不带 token 被拒绝"""
    response = client.get("/me")
    assert response.status_code == 401  # 或 403


def test_me_with_valid_token():
    """带正确 token 能通过"""
    # 先登录拿 token
    login_resp = client.post("/login", json={
        "username": "testuser", "password": "test123"
    })
    token = login_resp.json()["access_token"]

    # 带 token 访问受保护接口
    response = client.get("/me", headers={
        "Authorization": f"Bearer {token}"
    })
    assert response.status_code == 200
    assert "testuser" in response.json()["message"]


# ====== 测试业务逻辑 ======
def test_agent_chat():
    """测试 Agent 对话接口"""
    login_resp = client.post("/login", json={
        "username": "testuser", "password": "test123"
    })
    token = login_resp.json()["access_token"]

    response = client.post(
        "/agent/chat?message=35度等于多少华氏度",  # GET 参数用 params，POST body 用 json
        headers={"Authorization": f"Bearer {token}"}
    )
    assert response.status_code == 200
    assert "reply" in response.json()
```

**pytest 常用功能**：

```python
# ====== Fixture：共享初始数据 ======
@pytest.fixture
def auth_token():
    """每个测试都能用这个 fixture，不用重复登录"""
    client.post("/register", json={"username": "fixture_user", "password": "123"})
    resp = client.post("/login", json={"username": "fixture_user", "password": "123"})
    return resp.json()["access_token"]

def test_something(auth_token):
    """直接注入 token，pytest 自动调用上面的 fixture"""
    response = client.get("/me", headers={"Authorization": f"Bearer {auth_token}"})
    assert response.status_code == 200


# ====== Mock：不真的调用外部 API ======
from unittest.mock import patch

@patch("agent.call_weather_api")  # 替换掉真的 API 调用
def test_weather_agent(mock_api):
    mock_api.return_value = {"temp": 25, "status": "晴"}  # 返回假数据
    response = client.post("/chat", json={"message": "查天气"})
    assert "25" in response.json()["reply"]
```
